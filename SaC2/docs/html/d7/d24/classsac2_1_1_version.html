Detailed Description

Represents the version number of an under development project.

!

It allows to control the version of a project by comparing with another one. The format of the version number is: major.minor[.build[.revision]]. All these four numbers are 8-bit, therefore they must be between 0 and 255.

Note:
The following principle could be adopted for versioning:
  • MAJOR: significant changes in functionality, where backward compatibility cannot be assumed.
  • MINOR: new features or fixes have been added.
  • BUILD: changes of the platform, compiler.
  • REVISION a higher number might be used when minor bugs are fixed.
